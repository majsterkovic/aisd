Algorytmy z powracaniem¶

Celem laboratoriw bya implementacja algorytmw znajdowania cyklu Eulera oraz cyklu Hamiltona dla grafw nieskierowanych. Przygotowanymi przez mnie s:

  • Algorytm Robertsa-Floresa
  • Algorytm Hierholza

Do realizacji zadania naleao wybra dowoln reprezentacj grafu. Zdecydowaem si wykorzysta macierz ssiedztwa, ktra okazaa si najszybciej dziaajc reprezentacj (porwnanie zawarte jest w sprawozdaniu z algorytmw grafowych). W jej implementacji dokonaem modyfikacji tak, by bya dostosowana do grafu nieskierowanego. W macierzy ssiedztwa w czasie staym mona dodawa i usuwa krawdzie, a take sprawdza czy istnieje poczenie midzy nimi. Szczeglnie zoone nie jest te zwracanie nastpnikw.

Testowanie¶

Kody zostay przeze mnie opracowane w jzyku C++. Czas dziaania kadych operacji sprawdzam na 10 instancjach grafw o liczbie wierzchokw z przedziau 110-200. Przy tej wielkoci testuj instancje wypenione krawdziami w 30%, 50% oraz 70%. Jedynie kiedy szukam cyklu Hamiltona w grafie bez owego cyklu sprawdzam graf o liczbie wierzchokw $n = 20$. Test wykonuj 4 razy i bior redni z wszystkich otrzymanych wynikw. Do mierzenia czasu uywam funkcji biblioteki chrono przedstawionych poniej:

auto start = chrono::steady_clock::now();
auto stop = chrono::steady_clock::now();
auto duration = chrono::duration_cast<chrono::microseconds>(stop - start);
long long time_result = duration.count();

Algorytm Robertsa-Floresa¶

template<class GraphType>
bool Hamiltonian(int &start, int &v, int &visited, int &k, std::vector<int>&Path, std::vector<bool>&O, GraphType &G)
{
    std::cout << v << std::endl;
    O[v-1] = true;
    visited++;
    for(auto u : G.successors(v))
    {
        if(u == start && visited == G.order())
        {
            Path[k] = v;
            k++;
            return true;
        }
        if(O[u-1] == false)
        {
            if(Hamiltonian(start, u, visited, k, Path, O, G))
            {
                Path[k] = u;
                k++;
                return true;
            }
        }
    }
    O[v-1] = false;
    visited--;
    return false;
}

template<class GraphType>
bool Hcycle(GraphType &G, std::vector<int>&Path)
{
    std::vector<bool>O(G.order(), false);

    int start = 1;
    int visited = 0;
    int k = 1;
    Path[0] = 1;

    return Hamiltonian(start, start, visited, k, Path, O, G);
}

Algorytm Robertsa-Floresa rozwizuje problem poszukiwania cyklu Hamiltona, a wic jego zoono jest wykadnicza $O(n^2)$. Przeszukuje on graf oznaczajc kolejne wierzchoki jako odwiedzone i tworzc ciek. Jeli j zakoczy, a nie odwiedzi wszystkich wierzchokw funkcja zwraca fasz i powraca w poszukiwaniu alternatywnej cieki (odwiedza kolejnego nastpnika danego wierzchoka). Jeli j zakoczy i odwiedzi wszystkie wierzchoki to zwracana jest prawda, a wierzchoki s zapisywane do wektora Path.

Algorytm Hierholza¶

template<class GraphType>
std::vector<int> DFS_Euler(GraphType &G, int v, std::vector<int>&result)
{
    int size = G.order();
    for(int u = 1; u <= size; u++)
    {
        if(G.check_exist(u, v))
        {
            G.delete_edge(v, u);
            DFS_Euler(G, u, result);
        }
    }
    result.push_back(v);
    return result;
}

Zoono tego algorytmu okrelam jako $O(m)$, poniewa skada si on z ptli, ktra iteruje po wierzchokach tak dugo, jak istniej krawdzie. Mona by to te okreli jako $O(n^2)$. Algorytm Hierholza bada graf metod DFS i odkada wierzchoki na stos wnikowy kiedy dotrze do takiego, ktry nie ma nastpnikw. Nie ma tutaj spawdzenia czy graf jest eulerowski.

Podsumowanie¶

Oba algorytmy zachoway si zgodnie z analiz zoonoci obliczeniowej czas szukania cyklu Eulera przy pomocy algorytmu Hierholza nie rs tak szybko jak czas szukania cyklu Hamiltona przy pomocy algorytmu Robertsa-Floresa. Algorytm znajdowania cyklu Eulera mona by wzbogaci o funkcj bool isEulerian sprawdzajc najpierw czy graf spenia warunek istnienia cyku, czyli czy wszystkie wierzchoki maj stopie parzysty. Myl take, e warto byoby sprawdzi dziaanie tych funkcji na reprezentacji grafu w postaci listy nastpnikw, poniewa w algorytmach czsto z nich korzystamy.
